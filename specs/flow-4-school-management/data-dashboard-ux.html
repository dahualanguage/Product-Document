1 Navigation Flow

How does a user navigate between the three main views of the dashboard?
The dashboard has 3 screens: Overview (landing), Group/Cohort View, and Individual Student View. Users must be able to move between them without dead ends.

Screen Transition Map

FromActionTo
Dashboard OverviewClick any Summary CardGroup View (pre-filtered by card type)
Dashboard OverviewApply filters in Filter BarGroup View (filtered)
Dashboard OverviewSearch student nameIndividual Student View
Dashboard OverviewClick student in Risk ListIndividual Student View
Group ViewClick student rowIndividual Student View
Group ViewChange filtersGroup View (re-filtered)
Group ViewBack buttonDashboard Overview
Individual ViewSwitch tabSame screen, different tab
Individual ViewBack buttonPrevious screen (Overview or Group)
No dead ends: Every screen has a back path. The Individual View preserves the previous context — if the user came from Group View, “Back” returns to Group View with filters intact. If they came from Overview, “Back” returns to Overview.

7 Edge States & Empty States

What does the designer need to account for when there’s no data, partial data, or error states?
These states are often overlooked but are critical for a polished product. Every screen must handle them.
StateConditionUI Treatment
First-time empty School just onboarded, no students enrolled yet All Summary Cards show 0 / empty. Show illustration + message: “Enroll students and assign practice to start seeing data.” Link to Classroom Management.
Insufficient data Student has < 3 completed learning sessions Grey placeholder on all metric values. Trend lines hidden. Message: “Need at least 3 sessions to show trends.” Show progress bar: “2/3 sessions completed.”
No filter results Applied filters match 0 students Empty state in Group View: “No students match these filters.” Show active filter pills + “Clear all” button.
Tier change alert Student moves from Tier 2 â†’ Tier 3 (confirmed after 2 evals) Red badge dot on Risk Alerts card. Student appears in Risk List with “NEW” tag. Optional: browser notification if enabled.
Pending tier change Student met Tier 3 criteria once, waiting for 2nd eval Show “Pending” tag next to tier in Individual View. Tooltip: “Will be confirmed if criteria persist in next evaluation.”
Loading state Data is being fetched Skeleton placeholders (shimmer animation) on all cards and charts. Never show blank/empty during load.
Error state API call fails Inline error message per section (not full-page). “Unable to load data. Try again.” + retry button. Other sections still functional.

8 Responsive Considerations

BreakpointLayout Changes
> 1024px (desktop)4 Summary Cards in row. Side-by-side gauge + trend in Individual View. Full filter bar visible.
768px – 1024px (tablet)2 Summary Cards per row. Individual View tabs become scrollable horizontal strip. Filter bar collapses to “Filters” button with dropdown panel.
< 768px (mobile)1 Summary Card per row (stacked). Tabs become a dropdown selector. Charts reduce to simplified sparklines. Risk list becomes card-based layout.
Chart library note: Whichever chart library is used (Recharts, Chart.js, etc.), all charts must support touch interactions (swipe to pan, pinch to zoom on trend lines) and show tooltips on tap for mobile.
